PV:
	1、PV数据较以前最大的区别是现在iframe页面也是记录数据的，同时我们较以前丰富了一些新的客户端信息；

用户行为数据：

	1、前端脚本会在页面元素发生点击（click）、元素值发生变化（change）、表单被提交（submit）时记录此次变化的数据发送给数据采集服务以供用户行为分析；

	2、在以上行为发生时，前端脚本开始如下解析数据逻辑：

		若当前元素为忽略元素（tspan、text、g、rect、path、defs、clipPath、desc、title、use），则一直向元素的上级查找，直到找到非忽略元素。我们将找到的非忽略元素设置为我们的当前元素。

		接下来解析当前元素的路径，元素的路径作为元素在页面中区分其他元素的标识，其组成规则为单元素标识（如：元素TagName+元素ID+元素CLASS）递归组合其所有父元素标识（不包含body）的组合（如：/div#id.class/ul/li/a#id.class）。

		在解析当前元素路径时，若当前元素某一层父级元素为 A 或 BUTTON，我们会记录该父级元素的元素路径、元素位置、元素内容、元素链接地址。

		说明一下计算元素位置的规则，元素位置默认为1，若元素某一层父级元素为 TR 或 LI 时，我们将该父级元素在其父级元素下所有同类元素中的位置作为当前元素的元素位置。

		以下情况下的点击行为会被忽略：

			TEXTAREA、HTML、BODY 元素点击行为

			TYPE 不是 BUTTON、SUBMIT 的 INPUT 元素点击。

			不是 A、BUTTON、INPUT、IMG 时，且其包含的子元素层级大于4的情况。

		随后我们根据不同行为不同元素记录解析数据：

			将图片元素（IMG）的 src 属性值作为 h 属性记录

			将链接元素（A）的 href 属性值作为 h 属性记录

			将元素的 title 属性值作为 v 属性记录

			当点击到图片时，将图片的 alt 属性或图片名作为 v 属性记录

			当点击到输入框时，将输入框内容作为 v 属性记录

			当点击到非叶子元素时，将该元素的所有文本内容作为 v 属性记录

			在 type 为 radio、checkbox 的 INPUT 元素或 SELECT 元素值发生变化时，将元素值作为 V 属性记录

			在表单被提交时，将表单中 type 为 search、text、submit的 INPUT 元素的路径和值分别作为 x、v 属性记录

曝光：

	1、前端脚本会在页面加载完成后，获取页面所有到商品终极页、商铺页的链接元素。忽略从商铺页或商品终极页跳转到商铺页的链接，例如商铺页和商品终极页的导航链接。

	2、对这些链接元素根据链接地址排重后，将显示在屏幕内链接元素数据发送到数据采集服务，并且每个元素的数据只会发送一次。

	3、页面每次滚动，缩放，或有新的满足条件的链接元素被添加到页面时，都会重复上述操作。
